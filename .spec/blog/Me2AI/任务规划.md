# 任务规划

实现一个windows+mac+linux通用的插件  能实现选择某一段文本后 悬浮一个悬浮球 如果点击悬浮球 会对当前文本用agentic ai 去解决选择文本的问题

---

## 跨平台文本选择悬浮球 Agentic AI 插件

技术栈：Rust + Tauri，支持 Windows / macOS / Linux

### 阶段一：全局文本选择监听

- [x] 1.1 platform 层：分别实现 macOS（osascript 轮询）、Windows（SetWinEventHook）、Linux（xclip/xsel PRIMARY 选区轮询）的全局选中文本监听
- [x] 1.2 在 `platform/mod.rs` 统一抽象 `PlatformSelector` trait，`start_selection_listener` 封装事件推送
- [x] 1.3 获取选中区域的屏幕坐标（用于定位悬浮球），封装为 `SelectionEvent { text, x, y }`
- [x] 1.4 编写平台适配层的单元测试，覆盖创建/停止两种场景

### 阶段二：悬浮球窗口

- [x] 2.1 Tauri 端：float_ball 窗口设为无边框、置顶、透明背景（tauri.conf.json）
- [x] 2.2 后端 Command：`show_float_ball(x, y)` / `hide_float_ball()` / `show_result_panel()` / `hide_result_panel()`
- [x] 2.3 前端 UI（React）：FloatBall 组件，磨砂玻璃 + AI 图标，loading 旋转动画，点击触发 AI
- [x] 2.4 悬浮球自动触发：`useSelectionEvent` hook 监听 selection_event 并调用 showFloatBall
- [x] 2.5 窗口 command 单元测试（start_listening 返回 Ok）

### 阶段三：Agentic AI 问题解决核心

- [x] 3.1 `services/agent_service.rs`：AgentService + ReAct 循环（最大 max_iterations）
- [x] 3.2 支持 OpenAI / Anthropic / Ollama 三种 Provider，SSE 流式解析
- [x] 3.3 实现工具链：web_search（DuckDuckGo）、summarize；Tool trait 可扩展
- [x] 3.4 流式 chunk 通过 Tauri 事件（ai_chunk / ai_done / ai_error）推送到前端
- [x] 3.5 `repositories/history_repo.rs`：JSON 文件持久化，append/list（分页+最新优先）/clear
- [x] 3.6 集成测试：mockito mock OpenAI SSE、错误响应、CancellationToken

### 阶段四：结果展示面板

- [x] 4.1 result_panel 子窗口（Tauri），点击悬浮球 → 隐藏悬浮球 → 显示结果面板
- [x] 4.2 react-markdown + rehype-highlight 代码高亮，流式打字光标动画
- [x] 4.3 面板头部：复制 / 重新生成 / 关闭 三个操作按钮
- [x] 4.4 HistoryPage：分页卡片列表，点击展开全文，清空功能
- [x] 4.5 ResultPage + HistoryPage UI 完整实现

### 阶段五：设置与配置管理

- [x] 5.3 系统托盘图标：Enable/Disable、Settings、History、Quit 菜单项
