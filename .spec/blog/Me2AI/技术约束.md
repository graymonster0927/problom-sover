# 技术约束

## 技术栈

Rust+Tauri

## 代码规范

核心原则

   UI层
   ↓
   Tauri Command层
   ↓
   Service业务层
   ↓
   Repository / System层
   ↓
   系统或API

通用项目结构
my-tauri-app
│
├─ src/                       # 前端 (React / Vue / Svelte)
│   ├─ components/
│   ├─ pages/
│   ├─ hooks/
│   ├─ services/              # 调用 tauri invoke
│   ├─ store/                 # 前端状态管理
│   └─ main.ts
│
├─ src-tauri/
│   ├─ Cargo.toml
│   ├─ tauri.conf.json
│
│   └─ src/
│       │
│       ├─ main.rs            # 应用入口
│       ├─ lib.rs             # 公共模块
│       │
│       ├─ commands/          # Tauri命令 (UI调用)
│       │   ├─ mod.rs
│       │   ├─ app_command.rs
│       │   ├─ window_command.rs
│       │   └─ system_command.rs
│       │
│       ├─ services/          # 业务逻辑
│       │   ├─ mod.rs
│       │   ├─ translate_service.rs
│       │   ├─ clipboard_service.rs
│       │   └─ settings_service.rs
│       │
│       ├─ repositories/      # 数据访问层
│       │   ├─ mod.rs
│       │   ├─ settings_repo.rs
│       │   └─ history_repo.rs
│       │
│       ├─ models/            # 数据结构
│       │   ├─ mod.rs
│       │   ├─ translate.rs
│       │   └─ settings.rs
│       │
│       ├─ state/             # 全局状态
│       │   ├─ mod.rs
│       │   └─ app_state.rs
│       │
│       ├─ utils/             # 工具
│       │   ├─ mod.rs
│       │   ├─ http.rs
│       │   ├─ clipboard.rs
│       │   └─ logger.rs
│       │
│       ├─ platform/          # OS相关功能
│       │   ├─ mod.rs
│       │   ├─ windows.rs
│       │   └─ macos.rs
│       │
│       └─ errors/            # 统一错误
│           ├─ mod.rs
│           └─ app_error.rs
│
└─ README.md

## 测试

每个需求或任务或变动完成后必须有完整严谨测试

## 注意

1. 一定不要内存泄露
2. 三方库一定要稳定且可靠
3. 代码风格统一且精简规范, 具有可读性, 易于维护
